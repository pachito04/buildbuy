- **Type:** implementation (infrastructure)
- **Sequential after:** nothing (first task)
- **Spec requirements:** GAP1 DB Guard; GAP2 award_mode flag + pool_company_awards + triggers; GAP3 pool_providers table + RLS; GAP4 triggers + pool_companies_own_delete policy; GAP5 pool_number + chk_evento_tipo
- **File:** `supabase/migrations/028_pool_schema.sql` (NEW)
- **What:**
  1. `award_mode` column on `purchase_pools`: `ADD COLUMN IF NOT EXISTS award_mode text NOT NULL DEFAULT 'leader' CHECK (award_mode IN ('leader','per_company'))`.
  2. `pool_number` sequence + column + backfill ordered by `created_at, id` + `setval` + `DEFAULT nextval` + `NOT NULL` + `UNIQUE INDEX`. Exact pattern of migration 025 (`rfq_number`).
  3. `chk_evento_tipo` rewrite: DROP IF EXISTS + ADD CONSTRAINT with the 14-value set = the 13 from migration 024 + `'pool_joined'`. Do NOT use the 012 set.
  4. Verify whether `pool_requests` already has `UNIQUE(pool_id, request_id)`; if not, add it here (needed for `ON CONFLICT DO NOTHING` in 029's `pool_add_requirements`).
  5. Table `pool_company_awards` (grain: pool_id, company_id, rfq_item_id → winning_quote_item_id) with UNIQUE, indexes, and RLS policies: `pool_company_awards_member_select` (is_pool_member), `pool_company_awards_own_insert` / `own_update` / `own_delete` (is_pool_member AND company_id = caller's company).
  6. Table `pool_providers` (pool_id, provider_id, selected_by_company_id) with UNIQUE(pool_id, provider_id, selected_by_company_id), indexes, and RLS policies: `pool_providers_member_select` (is_pool_member), `pool_providers_own_insert` (is_pool_member AND selected_by_company_id = caller AND provider is own-or-global), `pool_providers_own_delete` (is_pool_member AND selected_by_company_id = caller).
  7. Policy `pool_companies_own_delete` on `pool_companies` (member can delete own row).
  8. Trigger function `pool_companies_link_guard` + trigger `trg_pool_companies_link_guard` BEFORE INSERT on `pool_companies`: allow self-join (company = pool owner), else require active company_links.
  9. Trigger function `purchase_pools_award_mode_lock` + trigger `trg_purchase_pools_award_mode_lock` BEFORE UPDATE on `purchase_pools`: reject award_mode change when OLD.pool_state ≠ 'borrador'.
  10. Trigger function `purchase_pools_state_guard` merged into or as second condition in the BEFORE UPDATE trigger: reject transition to 'cancelado' from 'cerrado' or 'cancelado'.
  11. Trigger function `pool_companies_withdraw_guard` + trigger `trg_pool_companies_withdraw_guard` BEFORE DELETE on `pool_companies`: reject delete unless pool is in 'borrador'.
  12. Rollback block commented at the bottom (DROP TABLE pool_providers, pool_company_awards; DROP TRIGGER + FUNCTION for each new trigger; ALTER TABLE purchase_pools DROP COLUMN award_mode, DROP COLUMN pool_number; DROP SEQUENCE; revert chk_evento_tipo to the 13-value set; remove pool_requests UNIQUE if added).
- **Manual verification checklist (record results in PR before merging):**
  - [ ] INSERT `purchase_pools` without `award_mode` → `award_mode = 'leader'` confirmed.
  - [ ] INSERT with `award_mode = 'per_company'` → succeeds; invalid value → CHECK violation.
  - [ ] UPDATE `award_mode` on pool in 'borrador' → succeeds; on 'confirmado' → trigger RAISES.
  - [ ] `pool_number` is NOT NULL on every insert; two concurrent inserts produce distinct values.
  - [ ] `pool_number` UNIQUE index prevents duplicate values.
  - [ ] INSERT `requerimiento_evento` with `tipo = 'pool_joined'` → succeeds.
  - [ ] INSERT `requerimiento_evento` with `tipo = 'consolidado'` → still succeeds (no regression).
  - [ ] INSERT `pool_companies` without active company_links (and company ≠ owner) → trigger RAISES.
  - [ ] INSERT `pool_companies` with active company_links → succeeds.
  - [ ] INSERT `pool_companies` for owner's own company → succeeds (self-join allowed).
  - [ ] Existing `pool_companies` rows pre-028 are untouched (BEFORE INSERT does not retroactively invalidate them).
  - [ ] DELETE `pool_companies` on borrador pool → succeeds; on confirmado pool → trigger RAISES.
  - [ ] UPDATE `pool_state = 'cancelado'` on 'cerrado' pool → trigger RAISES; on 'en_comparativa' pool → succeeds.
  - [ ] `pool_company_awards` SELECT: member sees all rows; non-member sees 0 rows.
  - [ ] `pool_company_awards` INSERT: member inserts own company → succeeds; wrong company → RLS blocks.
  - [ ] `pool_providers` INSERT: member inserts own provider or global (company_id IS NULL) → succeeds; provider of another company → RLS WITH CHECK blocks; wrong selected_by_company_id → RLS blocks.
  - [ ] `pool_providers` SELECT: member sees consolidated selection; non-member sees 0 rows.
  - [ ] `pool_providers` DELETE: member deletes own selection → succeeds; another company's row → blocked.

### T02 — [x] Write migration 029_pool_award_dispatch_rpc.sql

- **Type:** implementation (infrastructure)
- **Sequential after:** T01 (depends on pool_company_awards, pool_providers, award_mode, pool_number, pool_joined from 028)
- **Spec requirements:** GAP2 pool_finalize_award_mode_b; GAP3 pool_dispatch_providers; GAP5 pool_add_requirements
- **File:** `supabase/migrations/029_pool_award_dispatch_rpc.sql` (NEW)
- **What:**
  1. RPC `pool_dispatch_providers(p_rfq_id uuid) RETURNS int LANGUAGE plpgsql SECURITY DEFINER SET search_path = public`: resolves pool_id from rfqs; checks is_pool_member; reads DISTINCT active provider_ids from pool_providers where pool_id = v_pool_id; inserts into rfq_providers ON CONFLICT (rfq_id, provider_id) DO NOTHING; returns COUNT(*) from rfq_providers where rfq_id = p_rfq_id. GRANT EXECUTE TO authenticated.
  2. RPC `pool_finalize_award_mode_b(p_pool_id uuid) RETURNS void LANGUAGE plpgsql SECURITY DEFINER SET search_path = public`: checks is_pool_member; loads participating companies from pool_companies; maps pool_item_contributions → rfq_item_id per company; checks if every company has an award for every one of its rfq_items in pool_company_awards; if all complete AND pool_state = 'en_comparativa' → UPDATE pool_state = 'adjudicado'; otherwise no-op (idempotent). GRANT EXECUTE TO authenticated.
  3. RPC `pool_add_requirements(p_pool_id uuid, p_request_ids uuid[]) RETURNS void LANGUAGE plpgsql SECURITY INVOKER SET search_path = public`: guard for empty array; INSERT pool_requests SELECT ... ON CONFLICT DO NOTHING (RLS own_insert applies); read pool_number + jsonb_agg of company names; INSERT requerimiento_evento per request_id with tipo='pool_joined' and metadata {pool_id, pool_number, companies}. GRANT EXECUTE TO authenticated.
  4. Rollback block commented at the bottom (DROP FUNCTION pool_dispatch_providers(uuid), pool_finalize_award_mode_b(uuid), pool_add_requirements(uuid, uuid[])).
- **Manual verification checklist (record results in PR before merging):**
  - [ ] `pool_dispatch_providers` called by pool member → inserts deduped union of pool_providers (active providers only) into rfq_providers; returns correct count.
  - [ ] Re-execution for same rfq_id → count unchanged, no duplicates (ON CONFLICT idempotent).
  - [ ] `pool_providers` empty → COUNT = 0 returned; no rfq_providers rows inserted.
  - [ ] `pool_dispatch_providers` called by non-member → RAISES.
  - [ ] `pool_finalize_award_mode_b` with awards partial (not all companies done) → pool_state unchanged.
  - [ ] `pool_finalize_award_mode_b` with all companies' awards complete → pool_state = 'adjudicado'.
  - [ ] `pool_finalize_award_mode_b` called again after adjudicado → no-op (idempotent).
  - [ ] `pool_add_requirements` inserts pool_requests + one event per request_id; metadata contains pool_id, pool_number, companies.
  - [ ] `pool_add_requirements` with invalid tipo (simulate by temporarily removing 'pool_joined' from constraint) → transaction rolls back; pool_requests NOT partially written.
